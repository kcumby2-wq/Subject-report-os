# Daily Workflow — Run Every Session In Order

## STEP 1 — FEEDBACK CHECK (always first)

Open `feedback/feedback-log.md`. New entries?

Process each:
1. Identify which file it affects
2. Propose the edit to Kyron
3. On approval: make the edit, log in `feedback/improvements.md`

No new feedback? Move on.

## STEP 2 — THE 48-HOUR SLA GUARD

Check `data/orders/open-orders.md` for every open transcript order:

- Hours remaining against the 48hr clock?
- Anything inside 12 hours of breach → **FLAG AT TOP OF SESSION before any other work begins**
- New orders since last session? Confirm film link received, position, class year, contact info complete. Flag missing items.
- Delivered orders → move to fulfilled, trigger SM upsell sequence draft

## STEP 3 — DATA INTAKE

Check `data/` for new files since last session:

- New transcript in `data/transcripts/`? → run evaluation-processing skill → family-facing summary → save to `output/reports/` → check score: 4.0+ → flag as Active Profile candidate (consent check)
- New enrollment in `data/programs/`? → extract action items → update the relevant project file
- New competition or event? → map which athletes need evaluations → flag timing opportunities for outreach

## STEP 4 — GOALS PULSE

Open `goals/goals.md`. One line per goal: did anything move this forward? Anything blocked? Output a 6-line status pulse at the top of the session.

## STEP 5 — TODAY'S PRODUCTION

Based on goals and pipeline:

- Content day → write in SR voice → `output/content/`
- Course day → improve course portal content → `output/content/`
- Nurture day → email sequences per funnel stage → `output/sequences/`
- Report day → weekly or monthly report → `output/reports/`
- Upsell day → SM cross-sell sequences → `output/sequences/`

## STEP 6 — CLOSE THE LOOP

- Every deliverable saved to `output/` with date in filename
- Session log appended to `output/reports/session-log.md`
- List anything waiting on Kyron before ending

**Rule:** if a session ends and nothing new exists in `output/` the session failed. Files are the proof of work.
